ALMA-DIP

Reconstruct radio-astronomical images directly from ALMA visibility data using a Deep Image Prior U-Net generator, a Student-t data-fidelity loss for outlier robustness, and bootstrap resampling for per-pixel uncertainty maps.

How It Works #

Standard CLEAN-based imagers reconstruct the sky brightness distribution by iteratively subtracting the PSF from a dirty image. DIP takes a different approach: it parameterises the image as the output of a randomly initialised U-Net and fits the network weights directly to the measured visibilities via gradient descent. The network architecture itself acts as a structural prior — favouring smooth, physically plausible images — without ever being trained on labelled data.

The data-fidelity term uses a bivariate Student-t negative log-likelihood rather than the usual least-squares loss. This makes the reconstruction significantly more robust to outliers caused by radio-frequency interference (RFI) or calibration errors.

Uncertainty estimation is handled by running B independent reconstructions, each with visibility weights drawn from a Poisson(1) or Exp(1) distribution. The resulting per-pixel spread gives a principled measure of how well the data constrain each image pixel.

U-Net Generator (DIPUNet)

The generator is a 5-level U-Net that takes a fixed random tensor z ∈ ℝ^[1 × D × H × W] as input and outputs the reconstructed image I ∈ ℝ^[H × W]. Because z is never updated — only the network weights are — the network's implicit bias toward structured outputs acts as the prior.

Key design choices that differ from the original DIP paper: reflection padding (avoids edge artefacts), GroupNorm (stable on single-sample batches), bilinear upsampling (no transpose-conv checkerboard), and softplus output for positivity.

Stratified Visibility Sampler (VisibilitySampler)

For large datasets (N > 5 M), the memory path draws mini-batches from a mixture of five components to ensure good coverage of (u, v) space:

ComponentWeight (α)Description
Uniformalpha_uniformDraws any visibility with equal probability
Radialalpha_radialStratifies by baseline length ρ = √(u²+v²)
Inv-radiusalpha_inv_radiusWeights ∝ 1/ρ^γ, boosting short baselines
Weight-propalpha_weightSamples proportional to visibility weight w
Angularalpha_angularStratifies by position angle (optional)

Each drawn index is returned alongside its mixture density p_mix(i), which is used to compute the SNIS correction factor u_unif / p_mix(i) so the stochastic gradient remains an unbiased estimate of the full-data gradient.

Quick Start #

Place your visibility file (DG_Tau.txt) in the project directory and run:

python run_alma_dip_bootstrapping.pybash

For a quick test (runs in a few minutes on GPU):

from alma_dip_bootstrapping import DIPConfig, reconstruct_dip, pick_device
import numpy as np

data   = np.loadtxt("DG_Tau.txt", skiprows=1).astype("float32")
uv     = data[:, :2]
vis    = data[:, 2:4]
weight = data[:, 4]

cfg = DIPConfig(num_iters=200, out_every=50, tv_weight=1e-2)
device = str(pick_device())   # auto: MPS → CUDA → CPU

image, dirty, psf = reconstruct_dip(
    uv=uv, vis=vis, weight=weight,
    img_size=(540, 540),
    cfg=cfg, device=device,
)python

Input Data Format #

The visibility file is a whitespace-delimited ASCII table with a single header row followed by one visibility per line: 

u            v            Re           Im           weight
-12345.678   890.123      0.00412      -0.00231     245.3
...
txt
ColumnUnitDescription
uwavelengthsEast baseline coordinate
vwavelengthsNorth baseline coordinate
ReJyReal part of visibility
ImJyImaginary part of visibility
weightJy⁻²Natural (or Briggs) weight

All columns are cast to float32 after loading. The file is loaded with numpy.loadtxt(..., skiprows=1).

Alternatively you can pass a 5-column [N, 5] array directly to any of the reconstruction functions and omit the vis and weight arguments.

Output Files #

All outputs are written as 2-D single-precision FITS images (float32).

DG_Tau_dip_image.fits DIP reconstructed intensity map — the main science output.
DG_Tau_dip_dirty.fits Dirty image (weighted back-projection of the visibilities).
DG_Tau_dip_beam.fits Point spread function (PSF / synthesised beam).
DG_Tau_bootstrap_mean.fits Pixel-wise mean over the B bootstrap replicates.
DG_Tau_bootstrap_std.fits Pixel-wise standard deviation — 1σ uncertainty map.
DG_Tau_bootstrap_p16.fits 16th-percentile map (lower ±1σ-equivalent bound).
DG_Tau_bootstrap_p84.fits 84th-percentile map (upper ±1σ-equivalent bound).

Bootstrap Uncertainty #

Classical CLEAN does not naturally produce uncertainty maps. ALMA-DIP provides two bootstrap schemes that quantify how much each pixel is constrained by the data:

MethodWeight distributionInterpretation
poisson s_i ~ Poisson(1) Classical non-parametric bootstrap. Integer counts per visibility.
bayesian s_i ~ Exp(1) Bayesian bootstrap. Continuous weights, smoother replicate distribution.

The weights are computed without storing an N×B matrix — each weight is derived on-the-fly from a deterministic hash of the visibility index and replicate number (SplitMix64 mixing).

📋
The 16th and 84th percentile maps bracket a ±1σ-equivalent interval under a Gaussian distribution, making them directly comparable to image-plane noise estimates from CASA's tclean.

Computation Paths #

PathWhen selectedGPU memoryPer-iteration cost
Small N ≤ auto_threshold_n O(N × max(H,W)) Low (all trig precomputed)
Memory N > auto_threshold_n O(batch_vis × max(H,W)) Higher per step, SNIS-corrected

Small path

Precomputes the full N×W cosine/sine tables Cul = cos(2πul), Sul = sin(2πul), and the corresponding N×H tables for v×m. The 2-D DFT is then evaluated per iteration as two matrix multiplications. Fast per iteration but requires O(N × max(H,W) × 4) floats of VRAM. For N = 500 k, H = W = 540 this is roughly 2 GB.

Memory path

At each iteration draws batch_vis indices from VisibilitySampler and computes the partial DFT on-the-fly for just those baselines. A Self-Normalised Importance Sampling (SNIS) weight corrects for the non-uniform draw probability. GPU memory is bounded by the mini-batch size, making this path feasible for datasets with tens of millions of visibilities.

Device Selection #

from alma_dip_bootstrapping import pick_device

# Auto-detect (MPS → CUDA → CPU)
device = str(pick_device())

# Force a specific backend
device = str(pick_device("mps"))   # Apple Silicon GPU
device = str(pick_device("cuda"))  # NVIDIA GPU
device = str(pick_device("cpu"))   # CPU fallbackpython
⚠️
On MPS, torch.float32 is used throughout. MPS does not support float64 operations, so the bootstrap hash outputs are computed on CPU before being transferred.
